Critique and identify improvements for code provided code.
Look at codebase patterns, design consistency, structural alignment. 
Code structure and style should be clear, concise and maintainable. 

Focus on: 

 * Usage of common design patterns and best practices
 * Pythonic conventions and usage
 * No magic numbers
 * No global variables or state
 * No hard-coded values
 * Proper usage of Default argument values
 * Avoid or limit use of staticmethod and classmethod
 * Classes:
    * Use Properties where appropriate
 * Functions:
    * Structure and length
    * Complexity and nested looping layers
    * Logical flow
    * Modularity
* Exceptions:
    * Best practices for error handling
    * Proper Exception types
    * Custom Exceptions where appropriate
    * Clarity in exception messages
    * Caught and handled properly
